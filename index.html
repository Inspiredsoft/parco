PARCO.

PARCO è il sistema per la gestione, il monitoraggio ed il controllo dei rifornimenti di carburante del parco auto aziendale, effettuati attraverso l’utilizzo di "carte carburante" rilasciate da Q8 o Agip (ENI). Il sistema è attualmente utilizzato dall'azienda sanitaria reginale per le emergenze ARES118.

E' possibile accedere alla demo online di parco ospitata sulla piattaforma Pass di Red Hat OpenShift utilizzando le credenziali: username admin e password secret.

Funzionalità

  • Gestione anagrafica parco auto aziendale:
    • targa, stato, assegnazione, scadenza bollo/revisione, dotazioni (tomtom, etc.);
    • modello, capienza serbatoio, tipo carburante utilizzato, consumo urbano/extraurbano;
  • Gestione anagrafica carte carburante e carte pin:
    • numero carta, identificativo, associazione al mezzo;
    • associazione della carta pin alla perona (attualmente disponibile solo per Agip);
  • Acquisizione file dati sui rifornimenti generati dai sistemi di fatturazione di Agip e Q8;
  • Elaborazione dati relativi ai rifornimenti effettuati attraverso le carte carburante al fine di:
    • identificare rifornimenti anomali;
    • produrre report di consumo e spesa;
  • Verifica degli scontrini consentendo agli operatori (periferici) di inserire i numeri di scontrino e giustificare le anomalie rilevate.

Rilevazione Anomalie

Opportuni task asincroni vengono attivati a seguito dell’importazione di nuovi dati e al fine di rilevare eventuali anomalie. Le anomalie rilevabili sono:

  • Anomalie sul chilometraggio dichiarato;
  • Anomalie sulla frequenza dei rifornimenti;
  • Anomalie sul consumo (ciclo urbano/extraurbano);
  • Anomalie sull’utilizzo della carta carburante;
  • Anomalie sulla tipologia di carburante;
  • Anomalie sulla capienza del serbatoio;
  • Anomalia sullo stato di servizio del mezzo;
  • Anomalie sull'utilizzo della carta pin;

Report

Esempi di reportistica generata sono le seguenti:

  • Report sulle scadenze del mese (polizze o bollo);
  • Report sul consumo medio mensile di carburante;
  • Report aggregato per area del consumo complessivo ed importo complessivo;

Schema Funzionale

Requisiti di Sistema

  • JBoss 7.1.0
  • Java 1.6+
  • MySQL 5.5

Installazione e Configurazione

La procedura di installazione e configurazione si esegue in pochi semplici passi:

  • Configurare il driver MySql come modulo JBoss copiando il jar mysql-connector-java-5.1.20-bin.jar ed il file module.xml (presenti nella cartella res/mysql del progetto) nella cartella JBoss modules/com/mysql/jdbc.

  • Configurare nello standalone.xml di JBoss il driver MySql ed il datasource "parco" copaindo il contenuto dei file datasource.xml e driver.xml nella cartella nella cartella res/mysql del progetto.

  • Creare la base dati "parco" su MySql ed eseguire lo script parco.sql presente nella cartella res/db del progetto.

Importazione dati

Per importare i dati relativi al proprio parco auto o per modificare quelli esistenti è possibile utilizzare il file Excel anagrafica.xls presente nella cartella res/import del progetto. Attraverso tale file è possibile importare:

  • Le tipologie di mezzi
  • L'anagrafica dei mezzi
  • Le carte carburante
  • Le dotazioni dei mezzi
  • Le polizze assicurative
  • Le postazioni cui i mezzi sono associate (ed i relativi centri di costo)
  • L'anagrafica delle persone

Si noti che alcune delle informazioni sui mezzi e sulle carte carburante sono comunque generate (se non presenti) anche in fase di importazione dei file di consumo prodotti da Q8 ed Agip.

Analogamente, per importare i rifornimenti è necessario disporre dei relativi file prodotti da Q8 (formato Excel) o Agip (formato TXT). A titolo di esempio è possibile utilizzare il file ESEMPIO_CONSUMI_AGIP.TXT disponibile nella cartella res/import del progeto.
